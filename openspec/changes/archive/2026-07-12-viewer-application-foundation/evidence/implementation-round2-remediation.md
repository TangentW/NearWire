# Implementation Review Round 2 Remediation

Date: 2026-07-12

## Resolution Summary

| Review finding | Resolution | Evidence |
| --- | --- | --- |
| Cleanup receipt omitted claim-in-progress and already-cancelling attempts | Added a manager-owned cleanup registry and per-attempt cleanup owner before channel claim. Policy removal releases the capacity slot but retains cleanup ownership through core cancellation, late claim return, and direct late-channel cancellation. | Gated XCTest covers Pause, Reject, timeout, replacement, live stop, and claim-in-progress followed by stop. Every bounded wait times out while cancellation is held and the same receipt later completes after exactly one cancellation. |
| Handoff and shutdown were separate non-atomic callbacks | Replaced them with one handoff-owner protocol. Transfer occurs under the admission-manager terminal lock, shutdown closes transfer atomically, and the receipt awaits every accepted handle through `cancelAndWait()`. | XCTest covers ordinary automatic/manual handoff, accepted handoff with delayed shutdown cleanup, idempotent stop, and timeout/stop terminal competition. |
| Pending summaries could cross runtime generations | Each runtime now creates its own latest-only coalescer, deactivates it synchronously during stop, and guards delivery with the runtime token. The coalescer delivers one snapshot per MainActor turn and reschedules only when newer data exists. | Coalescer XCTest proves a MainActor heartbeat runs between snapshots, the latest snapshot arrives, and a deactivated generation publishes nothing. Application-model tests observe final states rather than sleeping. |
| Pre-Hello events could form an unbounded retained queue | Channel callbacks now synchronously enter the connection core's private serial queue. The existing bounded decoder completes or rejects the callback before Core requests more input. | A gated XCTest proves `receive` remains backpressured until Hello processing returns. Existing frame-size, protocol, timeout, 32-slot, and cleanup tests remain in force. |
| Terminal race coverage was incomplete and application tests used timing sleeps | Added a table-driven deterministic timeout-competitor test for Reject, Pause, replacement, stop, and channel termination in both winner orders. Replaced every application-model `Task.sleep` with status observation, listener callbacks, or explicit gates. | Viewer test source contains no wall-clock sleep except the injected live production scheduler. The full Viewer suite passes 53 tests with no failures or skips. |
| GeneralizedTime before 2050 was accepted | Canonical time parsing now validates the four-digit encoded year directly for GeneralizedTime and retains the 1950-2049 UTCTime rule. | XCTest verifies exact 2049/2050 tags, successful boundary parsing, and rejection of GeneralizedTime fixtures from 1949 and 2049. |
| Private-key reload required unsupported readable attributes | Creation still requests permanent and sensitive storage. Reload uses the exact application tag, key class/type, P-256 size, and actual nonexportability. It does not require login-Keychain `SecKey` reference attributes that are not reliably returned. | Real login-Keychain lifecycle and pure property tests pass; the active specification and operator documentation now state the portable validation boundary. |
| Keychain operations were not uniformly non-interactive and identity assembly had a broad fallback | Reads, exact identity lookup, and deletes now use an `LAContext` with interaction disabled. The default-Keychain `SecIdentityCreateWithCertificate` fallback was removed, and the exact returned identity's private key is checked against the owned key. | Real Keychain create/reload/reset passes. Injected identity assembly failure is attempted exactly once and fails closed. |
| Privacy manifest omitted direct UserDefaults use | Added `NSPrivacyAccessedAPICategoryUserDefaults` with reason `CA92.1` and corrected the spec, design, operator documentation, packaging assertion, and evidence wording. | App-hosted packaging XCTest parses the bundled manifest and requires exactly that category and reason. |

## Proportionality

No new validation script was added. Swift behavior and concurrency are covered by XCTest. Packaging, repository boundaries, and evidence capture continue to use the existing repository gate.
